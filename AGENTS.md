## OVERVIEW

Personal blog on Cloudflare Workers edge runtime. Hono SSR for public site + React SPA admin panel. D1 (SQLite) via Drizzle ORM, R2 for images, dual-language (zh/en).

## STRUCTURE

```
/
├── src/            # Cloudflare Worker — Hono app (SSR views + REST API)
├── admin/          # Separate React SPA — Vite + TipTap editor, builds to public/admin/
├── tests/          # Vitest with @cloudflare/vitest-pool-workers
├── migrations/     # D1 SQL migrations (generated by drizzle-kit)
├── public/         # Static assets (admin SPA output, CSS, favicon)
├── scripts/        # build.sh, seed.sh
├── hooks/          # Git hooks (custom core.hooksPath)
├── dev.sh          # ONLY sanctioned dev server manager
├── wrangler.toml   # Workers config (bindings, production env, domain: mrwuliu.top)
└── AGENT.md        # Dev workflow rules (dev.sh mandate)
```

## CONVENTIONS

- **No linter/formatter** — TypeScript type checking is the sole enforcement (pre-commit: `tsc --noEmit`)
- **Admin tsconfig is stricter** — has `noUnusedLocals`, `noUnusedParameters` (root does not)
- **ESM only** — both package.json files use `"type": "module"`
- **SSR views use `hono/jsx`** — `jsxImportSource: "hono/jsx"` in root tsconfig, NOT React
- **i18n routing** — zh at `/`, en at `/en/*`. Use `langPath(path, lang)` and `t(lang, key)`
- **DB access** — always via `createDb(c.env.DB)`, never instantiate drizzle directly
- **IDs** — `crypto.randomUUID()` everywhere, not auto-increment
- **Timestamps** — `datetime('now')` in D1 schema, ISO strings in application code
- **Route pattern** — each route file exports a Hono sub-router, mounted in `src/index.ts`
- **Zod validation** — API inputs validated via `@hono/zod-validator` (see `src/routes/posts.ts`)

## ANTI-PATTERNS (THIS PROJECT)

- **NEVER** use `npx wrangler dev` or `npm run dev` — use `./dev.sh start` only
- **NEVER** edit `public/admin/` — it's Vite build output, wiped on every build
- **NEVER** edit `migrations/` SQL files — generated by `drizzle-kit generate` from schema
- **NEVER** use React outside `admin/` — public site is `hono/jsx` SSR only
- **NEVER** add CSS files for public site — use Tailwind inline in JSX
- **NEVER** hardcode secrets — `.dev.vars` locally, `wrangler secret put` in prod
- **NEVER** bypass `createDb()` — all DB access goes through the factory
- **Pre-commit enforces typecheck** — no `as any`, `@ts-ignore`, or `@ts-expect-error`

## UNIQUE STYLES

- Dual-package single deploy: root Worker + admin SPA (separate npm projects, one deployment)
- Custom dev.sh manages Wrangler process lifecycle with PID tracking
- Git hooks via custom `hooks/` directory (`core.hooksPath`)
- Client-side i18n switching with inline `<script>` (no SPA router for public site)
- TipTap editor in admin with Markdown serialization + KaTeX math

## COMMANDS

```bash
# Dev server (ONLY way)
./dev.sh start        # Start on :8787 (auto-builds admin if needed)
./dev.sh stop         # Stop
./dev.sh restart      # Stop then start
./dev.sh migrate      # Apply D1 migrations locally
./dev.sh status       # Check running
./dev.sh logs         # Tail logs

# Testing
npm test              # All tests (API + UI)
npm run test:api      # API integration tests
npm run test:ui       # SSR render tests
npm run typecheck     # tsc --noEmit

# Database
npm run db:generate           # Generate migration from schema changes
npm run db:migrate:local      # Apply to local D1
npm run db:migrate:prod       # Apply to production D1
bash scripts/seed.sh          # Seed 20 test posts

# Deploy
npm run deploy                # Build admin + deploy to Cloudflare
git tag v1.0.0 && git push origin v1.0.0  # Trigger CI/CD

# Admin SPA only
npm run dev:admin             # Vite dev server (proxies /api → :8787)
```
