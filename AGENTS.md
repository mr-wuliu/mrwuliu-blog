## CONVENTIONS

- **No linter/formatter** ‚Äî TypeScript type checking is the sole enforcement (pre-commit: `tsc --noEmit`)
- **Admin tsconfig is stricter** ‚Äî has `noUnusedLocals`, `noUnusedParameters` (root does not)
- **ESM only** ‚Äî both package.json files use `"type": "module"`
- **SSR views use `hono/jsx`** ‚Äî `jsxImportSource: "hono/jsx"` in root tsconfig, NOT React
- **i18n routing** ‚Äî zh at `/`, en at `/en/*`. Use `langPath(path, lang)` and `t(lang, key)`
- **DB access** ‚Äî always via `createDb(c.env.DB)`, never instantiate drizzle directly
- **IDs** ‚Äî `crypto.randomUUID()` everywhere, not auto-increment
- **Timestamps** ‚Äî `datetime('now')` in D1 schema, ISO strings in application code
- **Route pattern** ‚Äî each route file exports a Hono sub-router, mounted in `src/index.ts`
- **Zod validation** ‚Äî API inputs validated via `@hono/zod-validator` (see `src/routes/posts.ts`)

## AI BLOG WRITING

AI agents can write blog posts directly via the production API. Credentials are in `.env` (gitignored).

### Authentication

All `/api/*` endpoints require either:
- **`X-API-Key` header** (for machine/AI access) ‚Äî value from `.env` ‚Üí `API_KEY`
- **Cloudflare Zero Trust identity** (for browser/admin access) ‚Äî automatic via Access login

AI must also bypass Cloudflare Zero Trust by sending Service Token headers:
- `CF-Access-Client-Id` ‚Üí from `.env` ‚Üí `CF_ACCESS_CLIENT_ID`
- `CF-Access-Client-Secret` ‚Üí from `.env` ‚Üí `CF_ACCESS_CLIENT_SECRET`
